Obligatory how to do a blogdown-like-website post

Last updated on 2019-06-24

I tend to repeat myself a lot, but I guess that there are many resources explaining how to do a website with blogdown. So, I will list all of the resources that I followed:

  1. blogdown, creating websites with R markdown, this book explains pretty much all the details. The chapter that I had to pay the most attention was the one about deployment of the website.

  2. Alison Hill’s Up & Running with blogdown, a graphical guide on how to set up the website, starting from the github repository, and ending on how to custom CSS, using formspree, or even adapting the domain to be .rbind.io

  3. Leslie Myint’s Tips for using the Hugo academic theme. This post was key to learn how to pick the right colors on the configuration of the site, and the code chunks

I used the Hugo Academic’s theme, which is more powerful than what I did for this site, but to keep things simple, almost all the files that need to be edited are in the content folder.

  • The main page is in content/authors/admin/_index.md

  • The posts are in content/post, and to generate a new post the blogdown’s addin does everything except writing a new post.

  • I used the content/project directory to keep the Software tab, and it is only necessary to create a folder for every software project, and edit one of the index.md files that come by default when creating an academic blog project. The picture was added as a file name featured.png in the same folder.

  • The content/publication works similarly, but we can also add a cite.bib / a pdf version of the paper, etc.

More details on how to configure the site are available in the academic site’s documentation, I hope this is useful!
